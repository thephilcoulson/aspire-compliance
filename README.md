# Aspire — Compliance Policy Framework

A comprehensive, industry-aligned compliance policy set for **Aspire** (aspireapp.com), the Southeast Asian business-finance / all-in-one finance operating system for growing businesses (business accounts, payments, corporate cards, expense management, and credit).

> ⚠️ **IMPORTANT — READ FIRST**
> These documents are **AI-generated templates** intended as a *starting framework*, not certification-ready or legally binding policies. Every policy must be reviewed and adapted by Aspire's legal, risk, and compliance functions, mapped to Aspire's actual systems and licences in each jurisdiction, and formally approved by an accountable owner before being relied upon.
>
> Do **not** present these to an auditor, regulator, or customer as-is. See [DISCLAIMER.md](DISCLAIMER.md).

## How to Use This Repository

1. **Assign owners** — each policy has a Document Control table; confirm the named accountable owner.
2. **Adapt, don't adopt** — tailor controls and values to Aspire's real environment, systems, and licences.
3. **Establish a review cadence** — annual review minimum; sooner on material change.
4. **Version control** — use this repo's git history and pull requests as the policy change log.
5. **Map to evidence** — link each control to operational evidence for audit readiness.

## Document Conventions

- Template values have been pre-populated with industry-standard defaults (effective date 1 January 2026, standard role-based owners, best-practice SLAs) that **must be validated** against Aspire's actual operations.
- "**shall**" — a mandatory control. "**should**" — recommended; deviations require documented risk acceptance.
